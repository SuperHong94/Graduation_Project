총알 

상황 
A 플레이어가 총알 발사 했다.
방법

1.서버가 모든 플레이어에게 A가 총알 쐈다고 알리고  
다른클라이언트에게 A 총알 하나 Active 하게 한다

장점:
서버에서 총알 좌표 위치 알 필요 없다.
구현 쉬워 보인다.
단점:
서버에서 총알 위치를 몰라서 충돌 체크 못함

2. A가 서버에 총알 발사했다고 알리고 서버에서는 A가 쏜 총알을 Active 한다.
(전제 Active한 총알들은 모든 클라이언트에게 해당 정보를 보내고 있어야함)

장점: 
서버에서 충돌체크를 할 수 있다.

단점:
서버에서 계속해서 총알 위치를 보내는것은 오버헤드가 크다.
그래서 몇초단위로 보낼텐데 그럴 경우 
클라에서는 총알의 움직임이 어색하지 않게 보간하는 작업을 해야한다.
